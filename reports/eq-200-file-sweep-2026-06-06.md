## 2. Method

- **Translate:** pre-built Release dll (`/Users/htd/Documents/Strata-smack/Tools/BoogieToStrata/Source/bin/Release/net8.0/BoogieToStrata.dll`) invoked directly via `dotnet <dll>` to skip per-call MSBuild restore. `--smack` flag set (required for SMACK Boogie because it sets `InferModifies = true`, populating `modifies` clauses absent from SMACK output). 8-way parallel via `xargs -n 1 -P 8`. Per-file 60s wall via `gtimeout 60`.
- **Verify:** Strata.exe with `--call-policy bodyOrContract --check-level minimal --inline-fuel 100`, post-fix htd/smack binary at `9d7132d72`. 60s wall budget per file. 8-way parallel.
- **Verdict map:**
  - rc=0 + "Finished with N goals passed, 0 failed" ‚Üí **PASS**
  - "Finished with N goals passed, M failed" with M‚â•1 ‚Üí **PARTIAL**
  - rc=3 + "Cannot find this fvar in the context!" ‚Üí **ELAB_FAIL**
  - rc‚àà{124, 143, 137} (gtimeout / SIGTERM / SIGKILL after ‚âà60s) ‚Üí **TIMEOUT**
  - rc‚àà{134, 139} or "stack overflow" / "segfault" ‚Üí **SO**
  - "‚ùì unknown" on a single failed goal ‚Üí **PASS_?** (none observed; rolls into PARTIAL when accompanied by goal-count line)
  - Otherwise ‚Üí **OTHER**
- **Translate phase:** 200/200 success in 49s; .err files all 0 bytes; .core.st files all non-empty. Size profile (.core.st): min 162 KB, p50 1.18 MB, p90 13.1 MB, max 61.7 MB.
- **Verify phase:** ~9 min wall (with mid-run timeout-budget interruption at the 174/200 mark; restart finished the remaining 26 in 130s).

## 6. Surprises (severity-ranked)

### 1. MEDIUM-HIGH ‚Äî ELAB_FAIL rate 28.5%, above 28-file Tier-1's 21.4% by +7.1 pp

- The 28-file ‚Üí 200-file is a 5.7√ó sample-size increase. PASS, PARTIAL, and SO rates moved by ‚â§2.5 pp (statistically stable) but ELAB_FAIL grew. Phase 4's bucket cross-tab attributes this to medium-bucket concentration: medium has the highest ELAB_FAIL rate (37.1%), and the 200-file sample is medium-skewed (48.5% medium vs the 28-file's smaller-skew).
- The single-class explanation holds (every ELAB_FAIL is the `old <X>` pattern); no new defect class.
- **Implication for audit's projection:** the audit's "~145 corpus-wide ELAB_FAIL" extrapolation (from 21.4%) is now low. Corrected projection at 28.5% is ~801 corpus files, **~5.5√ó the audit's number**. Lean4 PR #1331 (`old`-of-non-modifies-fvar) is correspondingly higher-leverage than previously stated.

### 2. LOW ‚Äî TIMEOUT rate 44.5%, above v2's bounded 9‚Äì26 projection

- This is a **stratification artifact, not a regression**: v2 ¬ß8.4 assumed 50/50 synth/jsm; this Phase 3 re-sample is 100% jsm, the corpus's TIMEOUT-heavy family. The 28-file Tier-1 had jsm-only TIMEOUT at 50.0% (39.3% + 10.7% TIMEOUT*); the 200-file 44.5% is **below** the 28-file rate by 5.5 pp. **Within-family rate is stable-to-improving as the sample widens.**
- Per-bucket: small 15.9%, medium 51.5%, large 72.5%. Size remains the dominant predictor (already known from Track B and v2).

### 3. NONE ‚Äî SO=0/200 confirms `balancedNondetIte` fix is corpus-stable

- Audit projected 336 pre-fix corpus-wide; post-fix expectation 0; measured **0**. The fix holds at 5.7√ó the prior validation set. The Option A solver-pin strip (`528fec0f2`) introduces no SO regressions.

### 4. NONE ‚Äî `e-15` evidence=0/200 confirms Decimal.lean:50 fix is corpus-stable

- Audit projected ~333 corpus files affected pre-fix; post-fix expectation 0; measured **0**.

### 5. ABSENT-DATA ‚Äî vacuous-PASS rate could not be re-measured

- The closeout's 81.2% vacuous-PASS finding (and the audit's 80%/85% jsm/synth split) cannot be confirmed or refuted from this single-policy sweep. **v2 ¬ß8.5 recommendation #1 (dual-policy run on the 22 deep-dived files) remains the right wedge**; this 200-file sweep does not substitute for it.
